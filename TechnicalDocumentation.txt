Technical Documentation (Architecture & Web App)
Our solution is built using a decoupled architecture, where the heavy lifting (data processing, SMOTE augmentation, and model training) is done in a secure backend notebook, and the outputs are served via an interactive Streamlit web application.

1. Technology Stack
Frontend / UI: Streamlit (Python)

Backend & Machine Learning: XGBoost, Scikit-Learn, Imbalanced-Learn (SMOTE)

Explainability (XAI): SHAP (SHapley Additive exPlanations)

Natural Language Processing: VADER (Sentiment Analysis), LLM API (Planned integration for natural language generation)

Data Manipulation: Pandas, NumPy

2. Core Application Modules
The web application (app.py) is divided into three main interactive modules to support HR decision-making:

A. Global Risk Dashboard
Functionality: Reads the pre-calculated risk_report_active.csv generated by our ML pipeline.

Features: Displays a searchable, filterable table of all currently active employees, highlighting those whose predicted risk of resigning exceeds a defined threshold (>15% High risk, >4% medium risk ,<4% Low risk).

B. Individual Employee Drill-Down & LLM Insights (WIP)
Functionality: Allows the HR manager to select a specific at-risk employee to understand the exact root causes of their flight risk.

Current State (SHAP): Visualizes the top positive and negative SHAP values from shap_active_df.csv, identifying specific metrics (e.g., DaysLateLast30, EmpSatisfaction) driving the score.

LLM Integration: We are integrating an LLM API to translate complex SHAP data and NLP sentiment scores into a natural, easy-to-read summary.

Example Prompt Architecture: The app will send the employee's anonymized top risk drivers to the LLM with the instruction: "Based on these SHAP values and a low sentiment score, write a 3-sentence summary of why this employee is a flight risk and suggest one HR action."

C. "What-If" Risk Simulator
Functionality: An interactive form allowing HR to create a "virtual employee" or modify an existing employee's parameters (e.g., increasing their salary, changing their manager, or simulating an absence).

Mechanism: The Streamlit app takes these user inputs, formats them into a dataframe, and passes them directly to the pre-trained XGBoost model's predict_proba() function to output a real-time predicted risk percentage.

Value Proposition: Enables proactive HR strategy by testing retention scenarios (e.g., "If I give this employee a 10% raise, how much does their flight risk drop?").

3. Data Flow & Security (Cybersecurity Pillar)
Offline Training: The ML model is trained securely offline. No real names or PII are used during the training phase.

Anonymized Payloads: The web app only interacts with anonymized IDs (EmpID) and aggregated risk scores.

LLM Safeguards: When querying the external LLM API for the drill-down explanations, strict system prompts are enforced to prevent prompt injection and ensure the LLM only uses the provided data without hallucinating.
